## ⭐ START HERE — the one open decision

We were mid-conversation on **making the map earned, not given** (Blaine's
idea). Current state: the corner **minimap (`K`)** shows the *whole* island for
free, while the journal fog-map only shows what you've walked. Blaine wants the
full-island view to be an **unlockable "attunement" to the island** — a
discovery beat, not a freebie.

**The agreed principle:** a map is *knowledge*. Both maps fog by default; the
whole island **blooms open when you attune**, per island, remembered in the save.

**Routes discussed (do more than one — variety is a pillar):**
- **Vista** — reach the open heights; the island lays itself out. Gives the
  high country (Highland/Scree/cliffs — mostly walled-off scenery today) a
  destination. *Not universal* (a lowland island has no heights).
- **Taste (berry)** — eat the island's fruit, "it knows you now." Universal +
  most poetic; **needs a small new "eat" verb**, which usefully opens the old
  **edibility** thread (what can I eat — discovered, not labeled).
- **Dwelling** — sleep a night at your camp; wake knowing its shape. Universal,
  **no new verb**, and finally gives the bedroll a *reason* (audit's "why a bed?").

**My recommendation (awaiting Blaine's nod on the routes):**
- **v1:** fog both maps; attune (persisted per island) via **vista OR dwelling**
  (together cover every island, no new machinery); a quiet reveal + a murmur;
  reveal *both* maps together so they never disagree.
- **v2:** the **taste/berry** route + the **edibility** system it anchors.

**Caveat raised to Blaine:** this reworks the minimap he just liked (it starts
fogged). He asked to think it over; **resume by confirming which routes**, then
build. Files it will touch: `src/game/main.ts` (the `drawMinimap` block near the
`drawOverview` fn, + a per-island `attuned` flag + the vista/`sleepToDawn`
hooks), `src/game/explored.ts` (the fog bitset, already queried by the journal),
`src/game/save.ts` (`SavedWorld.attuned?`), a reveal murmur in `src/game/murmurs.ts`,
and the almanac map in `src/render/journal.ts` for consistency.

## How this project is built

- **Build model:** Opus supervises; hard mechanical/architecture work goes to
  **Fable** subagents (`Agent`, `model: "fable"`, `isolation: "worktree"` when
  they'd collide on shared files). Opus verifies independently (`npm run check`
  + `npx vitest run` + `npm run build` + `npm run shot`), cherry-picks into
  master (auto-merge usually handles it), and `git worktree remove`s. Three
  Fable runs this session (design B, soil, audit) all landed clean.
- **Brief Fable in GAME vocabulary** (plants, critters, hue, glow, drift, sow,
  spread, palate, substrate, chain, soil) — **never** bio-manipulation words in
  prose; its classifier false-positives and returns empty.
- **Determinism is sacred:** ~18 pinned seeds stay bit-identical. New generation
  randomness comes from a **separate salted rng stream** (e.g. `makeRng(seed ^
  0x5b1e)`), never by inserting `rng()` into `generatePlantSpecies`'s main
  stream. No `Math.random`/`Date.now` in `src/life`, `src/world`, or scoring.
- **Commands:** `npm run dev` (play), `npm run check` (tsc), `npm run test`
  (vitest), `npm run build`, `npm run report` (md→html), `npm run shot --
  "seed=2438&chains=1" out.png 3000 960 640 "Escape,c"` (headless screenshot;
  keys are Playwright names; **run shots one at a time — fixed port**).
- **Keys:** `WASD/arrows` walk · `E` inspect · `Tab` menu · `C` living web ·
  `K` minimap · `Z` focus · `G` gather · `F` sow · `T` dig soil · `B` lay soil ·
  `Q` toss · `H` home/tend/sleep · `R` new island · `L` isles · `J` journal ·
  `M` murmurs · `P` postcard · `N` name · `?` help · `` ` `` dev readout · `Esc` close.
- **Toggles/dev-aids:** `wander.chains` (`?chains=0/1`, `?frontier`),
  `wander.minimap` (`K`); `?seed=`, `?at=tx,ty`, `?warm=N`, `?night=1`,
  `?lowtide=1`, `?aurora=1`, `?focus=1`, `?overview`, `?inspect`.
- **Docs & memory:** specs in `docs/superpowers/specs/`, plans in
